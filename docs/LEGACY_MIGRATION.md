# Legacy migration policy

PRMA succeeds `2025-Excool-Evermemory`, but legacy code and claims are not copied
into the trusted core without verification.

## Concepts retained

- Memory represented by a directed cyclic path
- Persistent learning in a shared neural substrate
- Directional competition and bounded synaptic strength
- Capacity, independence, robustness, and energy as research questions

## Changes in the v0 baseline

- A discrete hash encoder replaces sampled frequency encoding, eliminating hidden
  temporal aliasing from the baseline.
- One model instance learns all memories sequentially. Batch runs no longer reset
  weights while describing the process as accumulation.
- Configuration validation replaces interactive and hard-coded parameters.
- Recall and success are executable metrics rather than visual interpretation.
- Event cost is labeled as a proxy; it is not reported as physical energy.

## Import gate

A legacy component can enter `src/prma` only after:

1. its governing equation and array orientation are documented;
2. a small archived input/output fixture has a checksum;
3. a parity test reproduces the fixture within a stated tolerance;
4. computational complexity and memory use are measured;
5. its assumptions and failure conditions are added to the architecture document.

Legacy plots may be retained as historical artifacts, but they are not benchmark
evidence unless regenerated by a versioned PRMA experiment.
