# Research roadmap

## Phase 0 — engineering baseline (complete)

- Persistent sequential learning and deterministic encoding
- Capacity/interference benchmark and event-cost proxy
- Tests, CI, reviewed configurations, and provenance manifests

Exit criterion: a clean checkout reproduces the baseline result with one command.

## Phase 1 — faithful dynamical model

- [x] Specify membrane, propagation, learning, and saturation equations
- [x] Add explicit 1D and 2D spatial topology
- [x] Import the legacy model through parity tests against a checksummed fixture
- [x] Detect and test frequency-encoding alias periods
- [x] Add compact `O(T*N + N²)` trace recording
- [x] Define and test a frozen-weight partial-cue recall protocol
- [ ] Add an event-driven sparse execution engine with reference parity
- [ ] Define dynamical recall metrics and compare with the deterministic baseline

Exit criterion: mechanistic tests cover every equation and reproduce selected
legacy trajectories within documented tolerances.

## Phase 2 — robust memory evaluation

- Partial and noisy cue recall
- Retention and catastrophic-interference curves
- Capacity scaling over node count, path length, topology, and seed
- Conventional associative-memory and random-path baselines
- Bootstrap confidence intervals and preregistered success criteria

Exit criterion: every headline claim is generated by a versioned benchmark.

## Phase 3 — parameter optimization

- Multi-objective search over recall, interference, latency, and event cost
- Separate tuning and held-out evaluation suites
- Ablations for decay, competition, locking, replay, and topology
- Sweep resumption and experiment registry

This is the appropriate first meaning of “fine-tuning” for PRMA. Gradient-based
training becomes relevant only after a differentiable formulation is introduced.

## Phase 4 — model and hardware co-design

- PyTorch/JAX differentiable variant
- Spiking and asynchronous implementations
- Memristor/crossbar constraints and measured device parameters
- Energy estimates calibrated against a declared hardware process

Exit criterion: software proxies and hardware measurements are clearly separated,
with uncertainty propagated into comparisons.
